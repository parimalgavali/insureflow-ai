# Project Narrative

## Short Version

InsureFlow AI is a Guidewire-inspired P&C claims intelligence portfolio platform. It demonstrates policy and claim workflows, AI triage, document intelligence, RAG adjuster assistance, human review, audit, integration APIs, cloud readiness, quality gates, and observability using Java/Spring Boot, Python/FastAPI, Vue, PostgreSQL, Docker, and synthetic data.

## Medium Version

I built InsureFlow AI to demonstrate readiness for insurance technology and Guidewire-adjacent engineering roles. The project simulates a modern P&C claim workflow: policy and coverage validation, FNOL intake, claim lifecycle management, AI-assisted triage, document extraction, RAG-based adjuster support, human review, audit logging, integration APIs, deployment readiness, CI/security gates, and observability.

The system uses a Spring Boot backend, PostgreSQL/Flyway persistence, Python FastAPI AI services, local ML training, Vue 3 frontend, Docker Compose, GitHub Actions, Prometheus/Grafana, and Azure Container Apps templates. It uses synthetic data and is explicitly not an official Guidewire product or connector.

## Long Version

InsureFlow AI is a professional portfolio project built around a realistic insurance workflow rather than a generic CRUD demo. It models the journey from policy and coverage context through FNOL, claim creation, lifecycle updates, AI decision support, document intelligence, RAG-based adjuster assistance, human review, audit, integration events, deployment, and observability.

The backend is built with Java 21, Spring Boot, PostgreSQL, Flyway, JPA, validation, security, and integration tests. The AI layer is split into Python FastAPI services for rule-based and ML-backed triage, document intelligence, and retrieval-augmented adjuster assistance. The frontend is a Vue adjuster workbench designed to make the claim workflow and AI signals understandable to a reviewer. The platform layer includes Docker Compose, smoke tests, Azure Container Apps templates, GitHub Actions, Trivy scanning, coverage reports, Prometheus, and Grafana.

The project is Guidewire-inspired but not affiliated with Guidewire. Its purpose is to show that I understand insurance domain workflows, enterprise backend engineering, AI service boundaries, responsible AI controls, integration design, and cloud/platform readiness.
